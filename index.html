"Targeting" File Parsers with S2E and Kaitai Struct

 2017-10-23 |  #kaitai struct #s2e #symbolic execution

Introduction Recently I’ve been playing around with file parsers in S2E. This typically involves calling s2ecmd symbfile to make the parser’s input symbolic and then running S2E to explore different paths through the parser. However, this is a relatively heavy-handed approach; it makes the entire input file one big symbolic blob, which quickly causes path explosion. Additionally, we may only be interested in exploring paths that exercise specific functionality. So how can we achieve more targeted symbolic execution on file-based programs such as parsers?
